## v0.7.1 — Workflow Card Columns & Return Batch Page

- **Workflow card columns (prompt Section 14).** `buildWorkflowOrderCard()` now computes
  `total_quantity`, `total_cost_snapshot`, and surfaces `source_order_reference`
  (the OpenCart / source reference) as "Source / OC Ref". The card partial shows a
  Total Qty / Total Cost summary row and per-line cost (cost snapshot × qty = line total).
  Cost uses `supplier_cost_snapshot` from `ibs_order_items` — never selling price.
- **Return Batch create flow (prompt Priority C).** New `App\Domain\ReturnBatchReference`
  mirrors dispatch's DDMMYYYY pattern with an `R` prefix (R + DDMMYYYY, R + DDMMYYYY-P1
  for same-day repeats). `ReturnBatchWriteService::createBatch()` groups confirmed
  ("received") returns of the same supplier (max 50), inserts batch + batch items in a
  transaction, and marks each return `batched`. `listEligibleForBatch()` lists received
  returns not yet in a batch. New UI section "Create Return Batch" on /return-receive
  with checkboxes + a required confirmation. The existing owner-approve flow is unchanged.
- Stale "Return Batches (v0.5.1)" heading cleaned; status badges use the domain label.

## v0.7.2 — Return Batch → Payable Deduction Draft

- **Owner-gated deduction (prompt Priority D).**
  `PayableLedgerWriteService::createDraftFromReturnBatch()` creates ONE
  Return / Damage Deduction draft (ledger ref `RDB-<batchref>`, credit entry) from a
  return batch — but ONLY when the batch status is `owner_approved` and only once
  (idempotent on the batch reference). It does NOT post; the draft still requires the
  normal owner approval in the payable ledger before it affects the running balance.
- New "Return Batch Deductions" card on /supplier-payables lists owner-approved batches
  that don't yet have a draft, each with a "Create Deduction Draft" button.
- Route: `POST /supplier-payables/post-return-batch`.
- Version bump to 0.7.2 + VersionController feature bullets for 0.7.1 and 0.7.2.

# v1.8.3 — OpenCart Read-Only API Foundation

## Goal

Ship a read-only OpenCart connector for **staging.lokkisona.com** so IBS ERP can fetch real warehouse products and orders via JSON API.

**ERP workflow unchanged** — dispatch, return, payable, and sync import logic are not modified in this release.

### Routes (OpenCart 3.0.4.1)

```
/index.php?route=api/ibs/connection_test&api_token=...
/index.php?route=api/ibs/products&api_token=...&page=1&limit=20
/index.php?route=api/ibs/orders&api_token=...&page=1&limit=20
```

Optional order filters: `status_id`, `date_from`, `date_to` (YYYY-MM-DD).

### Product rules

- Join `{prefix}dispatch_location_product` where `from_warehouse = 1`
- Include product options/variants when present in standard OC tables
- Optional option images when `product_option_value_image` or POIP table exists

### Order rules

- Standard `oc_order` + `oc_order_product` + `oc_order_option`
- Courier/consignment fields read from existing columns when present (configurable map in `ibs_api.php`)
- No supplier-handled filter in OpenCart — ERP applies status mapping

## ERP configuration

Set in **System → Sync/API Settings** or `config/opencart.local.php`:

```php
'source_mode' => 'staging',
'enabled' => true,
'demo_mode' => false,
'api_base_url' => 'https://www.staging.lokkisona.com',
'api_key' => 'SAME_AS_OC_system_config_ibs_api_php',
'product_api_route' => 'api/ibs/products',
'order_api_route' => 'api/ibs/orders',
'max_rows_per_page' => 20,
'dispatch_bridge_required' => true,
```

See also `config/opencart.staging.example.php`.

## How ERP calls the API

`OpenCartReadClient` builds GET URLs (unchanged):

```
{api_base_url}/index.php?route={product_api_route}&api_token={key}&page={n}&limit={n}
```

- **Test Connection** (Sync/API Settings) calls product page 1 and checks `bridge_available`
- **Product preview** paginates `api/ibs/products`
- **Order preview** paginates `api/ibs/orders`

## Staging deploy order

1. Upload OpenCart package to staging.lokkisona.com
2. Copy `ibs_api.example.php` → `system/config/ibs_api.php`, set token
3. Postman tests (`packages/.../TEST-PLAN.md`)
4. Configure ERP Sync/API Settings
5. Test Connection → Sync Preview E2E

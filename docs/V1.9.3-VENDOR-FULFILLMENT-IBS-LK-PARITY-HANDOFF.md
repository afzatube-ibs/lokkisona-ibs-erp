# v1.9.3 — Vendor Fulfillment IBS-LK Parity Handoff

**Version:** 1.9.3  
**Release label:** Vendor Fulfillment IBS-LK Parity  
**Route:** `/order-workflow`  
**Baseline:** v1.9.2 Vendor Fulfillment Workflow Foundation  

## Summary

Rebuilds old IBS-LK Vendor Fulfillment UX inside the standalone ERP: compact stage cards, unified toolbar with **one status-aware bulk action** (never all four at once), founder-specific modals, 3-dot row menu, Created Report locked rows, and green success toast. IBS workflow remains independent from OpenCart — no OpenCart writes.

## Visual QA reference

Use uploaded **orderflow.mp4 and screenshots** side-by-side with live `/order-workflow` during acceptance. Capture screenshots under `docs/screenshots/v193-vendor-fulfillment-*.png` before commit approval.

## What changed

### UI
- Compact fulfillment stage cards + exception chips (`.vf-status-cards`)
- Toolbar: search, selected count, single bulk forward button, Hold/Cancel, per-page 20/50, Clear, owner Test Sync link
- Advanced filters collapsed (supplier, courier, date range)
- Product card: **No option selected** chip when no variant/options
- Created Report rows: batch ref, View Report link, locked info text
- Green toast (`.vf-toast-success`) after successful actions

### Modals
1. **Print & Move to Processing** — bulk packaging with totals + invoice tab
2. **Dispatch Report Created** — post bulk dispatch with report totals and links
3. **Confirm Hub Return Receive** — checklists + return note
4. **View timeline** — lazy load from `GET /order-workflow/history?id=`
5. **Add note** — `POST /order-workflow/note`

### Backend
- `OrderWorkflowStatus::rowActionLabel()` vs bulk labels (e.g. row **Start Packaging**, bulk **Print & Start Packaging**)
- `OrderWorkflowBulkWriteService`: homogeneous status/supplier validation; `bulk_hold`, `bulk_cancel`; `selectionTotals()`
- `OrderWorkflowWriteService::recordNote()`
- `DispatchReportRepository::findIncludedOrderMeta()` for View Report links
- Routes: `GET /order-workflow/history`, `GET /order-workflow/selection-preview`, `POST /order-workflow/note`

### Bulk rules (mandatory)
| Filter / homogeneous status | Single bulk button |
|----------------------------|-------------------|
| New Order | Bulk Receive Order |
| Order Received | Print & Start Packaging |
| Packaging | Mark as Shipped |
| Shipped | Create Dispatch Report |
| Mixed / other | No forward bulk (Hold/Cancel when eligible) |

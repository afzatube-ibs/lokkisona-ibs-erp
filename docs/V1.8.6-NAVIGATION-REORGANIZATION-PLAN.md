# IBS-LK Business Manager — v1.8.6 Navigation Reorganization Plan

**Status:** Awaiting approval — do not implement until signed off  
**Release target:** `1.8.6` — Navigation Reorganization  
**Prior release:** `1.8.5` Supplier Product Control Completion

---

## Goal

Clean the left menu into business departments and remove confusing sync/mapping clutter from the daily menu.

**Constraints for this build:**

- Reorganize menu visibility only — do not delete routes
- No OpenCart writes
- No order workflow logic change
- No product sync write logic change
- No migration unless absolutely required (prefer none)

---

## 4. Product Control sync action placement

**Rule:** Product Sync / Import / Reset must stay inside Product Control page.

### Placement

Add collapsible **"Product Sync / Import"** panel on `resources/views/product-control/index.php`, below page header (owner/admin, gated by `sync_preview.view` + `sync_preview.manage`).

### Content (extract to shared partial)

New file: `resources/views/partials/product-control-sync-panel.php`

Include existing markup from `sync-preview/index-head.php`:

- Connection & Product Sync Status
- Product Sync Diagnostics
- Product Sync Preview (load, paginate, import)
- Product Sync Reset form

### Header changes

Remove outbound `Sync Products` / `Sync Preview` buttons linking to `/sync-preview`. Replace with expand/collapse for embedded panel.

### Controller (read/presentation only)

`ProductControlController` loads same read services as `SyncPreviewController` product section:

- `ProductSyncReadService::status()`
- `ProductSyncDiagnosticsService::analyze()`
- Product preview session
- `WriteGate::productSyncImport()`

POST endpoints unchanged: `/sync-preview/preview-products`, `/sync-preview/import-products`, `/sync-preview/reset-product-sync`. Reset `redirect_to` becomes `/product-control`.

### Standalone `/sync-preview`

Keep route for order sync section + backward-compatible bookmarks. Add relocation banner: *"Product sync has moved to Product Control."*

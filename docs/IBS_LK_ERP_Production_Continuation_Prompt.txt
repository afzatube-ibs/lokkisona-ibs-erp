=== SECTION 4: BUSINESS MODEL SUMMARY ===
================================================================================

Lokkisona / Sonamoni sells products online.
Some orders are handled by supplier Iqbal & Brothers (supplier-handled orders).
Supplier sends products directly to customer/courier for those orders.
Lokkisona manages: order source, customer communication, courier, final approval,
accounts, reporting, and settlement.

ERP handles SUPPLIER-RELATED ORDERS ONLY — not all ecommerce orders.

Core business flow:
  1. Order comes from Lokkisona/OpenCart/PIT status mapping OR manual/offline entry.
  2. ONLY supplier-handled orders enter IBS-LK ERP (status mapping gate).
  3. Supplier receives order (Order Received).
  4. Supplier packs and ships the parcel.
  5. Owner/admin creates daily dispatch report.
  6. Dispatch report LOCKS supplier product cost snapshot (immutable).
  7. Supplier payable is based on dispatch snapshot — NOT selling price, NOT live cost.
  8. Courier/status mapping later updates Out For Delivery and Delivered.
  9. Returns are received and routed: supplier return OR owner warehouse return.
 10. Supplier ledger/settlement: payable, deductions, payments, balance.

Channel-neutral architecture: manual, offline, ecommerce, marketplace orders share
the same workflow when they are supplier-handled.

=== SECTION 6: MAIN FULFILLMENT WORKFLOW ===
================================================================================

Canonical path:
  New Order
  → Order Received
  → Packaging
  → Shipped
  → Dispatch Report Created / Created Report
  → Out For Delivery
  → Delivered

Stage meanings:
  New Order        = synced/mapped or manually entered for supplier fulfillment
  Order Received   = supplier accepted / product available
  Packaging        = supplier checked product, packing; print/packing around here
  Shipped          = parcel packed and handed to courier
  Dispatch Report  = owner created daily batch; cost snapshot LOCKED
  Out For Delivery = courier/PIT/status mapping — NOT supplier manual action
  Delivered        = terminal success; payable still from dispatch snapshot only

--- STAGE ACTIONS (enforced in OrderWorkflowWriteService + UI) ---

New Order:
  - Button: Receive Order
  - Also: Hold, Cancelled (with required note)
  - Supplier must NEVER move order back to New Order

Order Received:
  - Button: Start Packaging
  - Checkbox confirmation required (staff confirms products checked)
  - Do NOT show Ship button here

Packaging:
  - Button: Mark as Shipped
  - Checkbox confirmation required
  - After Shipped, order eligible for daily dispatch report

Shipped:
  - Delivery Stop (manual, note required) — from Shipped only
  - Create Dispatch Report: done on /dispatch-reports page (NOT workflow button
    when dispatch module ready — redirect note shown on workflow card)
  - Without dispatch report, order must not advance to payable

Dispatch Report Created / Created Report:
  - No supplier manual courier buttons
  - Courier/PIT mapping moves to Out For Delivery / Delivered later
  - Must remain visible in Order Workflow as Created Report stage

Out For Delivery / Delivered:
  - Source/courier mapping only — no supplier manual buttons

--- EXCEPTION WORKFLOW ---

Hold:
  - From New Order, Order Received, Packaging only
  - Requires reason/note
  - Resume Order → Order Received or Packaging (NEVER back to New Order)
  - Cancelled also allowed from Hold

Cancelled:
  - Before completion from New Order, Order Received, Packaging, Hold
  - Requires confirmation + reason. Terminal.

Delivery Stop:
  - Manual ERP action from Shipped only
  - Requires note/reason + confirmation
  - Flow: Shipped → Delivery Stop → Hub Return

Hub Return / Courier Return:
  - Shipped → Delivery Stop → Hub Return → Return Received confirmation
  - Supplier confirms return received at /return-receive

Customer Return / Order Returning:
  - From PIT/OpenCart return status mapping (Returning / Returning Warehouse)
  - Owner decides: Supplier House OR Lokkisona/Owner Warehouse destination
  - Supplier confirms received only for supplier-side returns

=== SECTION 8: DISPATCH REPORT WORKFLOW ===
================================================================================

Dispatch Report = official supplier payable checkpoint.
Supplier payable starts from Dispatch Report, NOT from Delivered.

Dispatch report must:
  - Include SHIPPED orders only
  - Create daily batch/report
  - Lock cost snapshot at creation time (IMMUTABLE — never recalculate old reports)
  - Move orders Shipped → dispatch_report_created
  - Write workflow history: "Dispatch Report {reference}"
  - Prevent duplicate inclusion of same order
  - Show report detail: orders, customers, qty, cost snapshot, status, date
  - NOT auto-create payable ledger (owner posts from /supplier-payables)

Reference format:
  DDMMYYYY base
  Same-day multiples: DDMMYYYY-P1, DDMMYYYY-P2, DDMMYYYY-P3

Cost snapshot rule (CRITICAL):
  If product cost was 500 yesterday and 600 today:
    Yesterday's dispatch report stays 500 forever.
    Today's report uses 600.
  Never use selling price or order total for supplier payable.

Route: /dispatch-reports — POST /dispatch-reports/create
Service: DispatchReportWriteService

=== SECTION 9: SUPPLIER PAYABLE / LEDGER MODEL ===
================================================================================

Supplier Account / Payable Ledger = financial heart of ERP.

Formula:
  Opening Balance
  + Product Cost Payable (from dispatch snapshot)
  + Supplier Invoice
  + Additional Payable / Extra Cost
  + Debit Adjustment
  - Return / Damage Deduction (after receive + owner approval)
  - Payment Made to Supplier
  - Advance Received from Supplier
  - Credit Adjustment
  = Net Payable / Balance

Ledger entry types:
  1. Product Cost Payable
  2. Supplier Invoice
  3. Additional Payable
  4. Return / Damage Deduction
  5. Payment Made to Supplier
  6. Advance Received from Supplier
  7. Debit Adjustment
  8. Credit Adjustment

Settlement workflow (v0.5.9 built):
  Draft → Prepared → Owner Approved → Paid → Closed

Settlement period options:
  Daily, Weekly, 1–15, 16–30/31, Monthly, Custom Date Range

Routes:
  /supplier-payables — ledger entries, dispatch post, approve/reject
  /settlements — period prepare, approve, mark paid, close
  /reports — 15 owner reports with print/CSV

Return deduction: ONLY after return receive confirmed AND return batch owner
approved AND explicit payable draft approval — NOT automatic on receive.

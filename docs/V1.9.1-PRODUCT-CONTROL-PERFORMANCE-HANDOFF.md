## What changed

### Performance
- **Listing:** SQL `LIMIT 20 OFFSET` — no loading 200 products / 500 variants on every GET
- **KPI chips:** Single aggregate SQL query; cached 60 seconds (`storage/cache/product-control/`)
- **Modal:** Lazy load via `GET /product-control/workspace?id=` — no embedded workspace JSON in page HTML
- **History:** Lazy load via `GET /product-control/history?id=` when History tab opened
- **Page load:** Local ERP DB only — removed `ProductSyncReadService::status()` / `testConnection()` on listing

### Snapshot UX
- **Snapshot last refreshed** label from `MAX(last_synced_at)`
- **24h stale warning** when catalog snapshot is older than 24 hours
- Copy updated: no automatic refresh; **Refresh Products** is manual owner action only

### Dev diagnostics (local env only)
- Timing footer: list query, KPI, variant aggregate, total ms
- `error_log` timing for index, workspace, and history endpoints

### Optional migration
- `database/migrations/0013_product_control_list_indexes.sql` — indexes on `supplier_model`, `source_model`, `last_synced_at`, variant `supplier_model` (manual apply)

### Unchanged
- Re-sync preserves supplier fields (`SyncPreviewWriteService` / `ProductWriteService`)
- No OpenCart writes, no Order Workflow changes
- Save still POST `/product-control/workspace/save`

# v1.8.2 — Sync/API Settings Foundation

## Goal

Configure OpenCart read-only sync without hardcoded staging URLs in code. Switch between Demo, Staging (`https://www.staging.lokkisona.com`), and Live (`https://www.lokkisona.com`) from the ERP UI.

## Where

**System → Sync/API Settings** (`/sync-api-settings`)

## Storage (no DB migration)

| File | Git | Purpose |
|------|-----|---------|
| `config/opencart.php` | Committed | Safe defaults, demo data |
| `config/opencart.local.php` | **Gitignored** | Server URL, API key, mode overrides |
| `config/opencart.local.example.php` | Committed | Copy template (no real keys) |
| `config/opencart.staging.example.php` | Committed | Staging reference values |

Bootstrap merges `opencart.local.php` over `opencart.php` at runtime.

## Settings fields

- Source Mode: Demo / Staging / Live
- Source URL
- API key / token (password field; **never shown after save**)
- Product API route
- Order API route
- Read-only mode lock (always on)
- Max rows per page: 20 (fixed)
- Product Sync Enabled
- Order Sync Enabled
- Dispatch Location Bridge Required
- **Test Connection** button

## Product sync help block

On `/sync-preview`, **Product Sync Help** diagnoses:

1. Connection problem
2. Bridge missing
3. No `from_warehouse` products
4. API route missing
5. Product option/variant not loaded
6. Missing required ERP table/column

Each issue includes a **Fix** line pointing to Sync/API Settings or Dev DB Activation.

## Safety

- No OpenCart write
- Read-only sync only
- No CREATE/ALTER on page load
- No migration in this build
- API keys must not be committed to Git

## Staging setup

1. Copy `config/opencart.local.example.php` → `config/opencart.local.php` on server
2. Open **System → Sync/API Settings**
3. Set Source Mode = Staging, URL = `https://www.staging.lokkisona.com`, API key, product route
4. Save → Test Connection
5. `/sync-preview` → load product preview → confirm import

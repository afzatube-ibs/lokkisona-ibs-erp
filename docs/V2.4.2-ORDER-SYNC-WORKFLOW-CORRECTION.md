## Why this build exists

Ledger and payable posting cannot be trusted until OpenCart sync and IBS workflow boundaries are correct. v2.4.2 enforces the same rules as the legacy **Vendor Fulfillment** extension (`ibs_lk_supplier_management_erp_v5.8.3` action logic): OpenCart feeds orders **up to Shipped**; IBS owns everything after.

## Final workflow rule

### OpenCart sync (import only)

| Rule | Implementation |
|------|----------------|
| Mapping sets **initial** IBS status only | `OrderSyncMappingRules` — allowed targets: New, Received, Packaging, Shipped, Hold, Cancelled |
| **Never** map to post-shipment statuses via sync | `dispatch_report_created`, `out_for_delivery`, returns blocked in mapping + preview |
| Existing orders: snapshot refresh only | `SyncImportWriteService::updateOriginSnapshot()` — never touches `ibs_status` |
| After **Shipped**, IBS workflow owns the order | `OrderSyncWorkflowBoundary::SHIPMENT_CEILING = shipped` |

### Supplier workflow (IBS actions)

```
New Order → Order Received → Packaging → Shipped
```

From **Shipped** (pre-dispatch):

```
Shipped → Delivery Stop → Hub Return Confirmed (hub_return)
```

- No dispatch report  
- No payable  
- No ledger  
- No return report batch (receive record + history only)

### Create Dispatch Report (financial lock)

```
Shipped → Create Dispatch Report → dispatch_report_created
```

- Locks `product_cost_snapshot` per order in `ibs_dispatch_report_items`  
- Order cannot move back to supplier phases  
- Payable draft may be created (existing v0.4.7 flow) — **no auto ledger posting in v2.4.2**  
- Demo/test orders blocked from dispatch (`OrderDemoGuard`)

### Post-dispatch return (Return Report)

```
Dispatch Report Created → Out For Delivery → Customer Returning → Supplier Confirmed → Return Report
```

- Return receive for `customer_return_to_supplier` **requires** dispatch item  
- Creates return report batch with locked dispatch snapshot adjustment  
- Hub return **blocked** after dispatch — must use customer return path

## Manual validation checklist

1. **Mapped OC order syncs** — Status Mapping row (e.g. Follow Up → New Order) → Test Order Sync → import → order at mapped status only  
2. **Shipped by mapping/action** — Map Processing → Shipped OR use Mark as Shipped action  
3. **Pre-dispatch Hub Return** — Shipped → Delivery Stop → Confirm Hub Return → receive confirm → **no** return report batch  
4. **Create Dispatch Report locks** — Shipped batch → Created Report → cannot revert to Packaging/Shipped  
5. **Post-dispatch return** — After dispatch → Customer Returning → receive → return report batch with dispatch snapshot amount  
6. **Demo blocked** — Demo sync order cannot enter dispatch batch  
7. **Re-sync existing** — Change OC status on dispatched order → IBS `ibs_status` unchanged, snapshot fields update

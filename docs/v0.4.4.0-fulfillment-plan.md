## Business Rules

### Dispatch gate (temporary manual)

- **Dispatch Report Created remains a required gate before Out For Delivery.**
- For v0.4.4.0: `shipped` → `dispatch_report_created` is a **manual/admin workflow action** (not dispatch table automation).
- **Action note is required** when moving to `dispatch_report_created` (owner should enter dispatch reference / note in `action_note`).
- `out_for_delivery` is **only** allowed from `dispatch_report_created` (not from `shipped`).
- UI must show DEV note near Dispatch Report Created action:

  > *Temporary manual dispatch gate. Product-wise dispatch report will be activated after dispatch migration/table setup.*

- Later sprint: replace manual gate with Daily Dispatch Report module after migration/table activation.

### Notes required

| Transition to | Note required? |
|---------------|----------------|
| `cancelled` | **Yes** |
| `delivery_stop` | **Yes** |
| `hub_return` | **Yes** |
| `order_returning` | **Yes** |
| `dispatch_report_created` | **Yes** (dispatch reference / note) |
| All other transitions | Optional |

### Audit

- Every action must insert `ibs_order_workflow_histories` (`from_status`, `to_status`, `action_note`, `changed_at`).
- `changed_by` populated from current authenticated user when available (`Auth::user()`).

### Manual order mirror

- Mirror `ibs_manual_orders.ibs_status` **only** when order came from manual order bridge.
- **Safe link today:** shared `order_reference` between `ibs_orders` and `ibs_manual_orders` (no `manual_order_id` on `ibs_orders`).
- Lookup: `ibs_manual_orders.manual_order_reference = ibs_orders.order_reference`.
- If no match, skip mirror silently.
- Do **not** change `ManualOrderWriteService` create logic.

### Legacy alias support (dev data)

Normalize on read/transition for one release:

| Legacy code | Maps to |
|-------------|---------|
| `confirmed` | `order_received` |
| `processing` | `packaging` |
| `ready_for_dispatch` | Block or document manual advance to `shipped` |

## Dispatch Gate — How It Works Without Dispatch Tables

```text
1. User: packaging → shipped
2. User: shipped → dispatch_report_created
   (manual action; note/reference required — e.g. "DR-04062026-12")
   → ibs_orders.ibs_status updated
   → ibs_order_workflow_histories row written
3. User: dispatch_report_created → out_for_delivery  ✓ allowed
4. User: out_for_delivery → delivered  ✓ allowed

Blocked:
- shipped → out_for_delivery  ✗ (must pass dispatch_report_created first)
```

No query to `ibs_dispatch_reports`. Gate is **pure workflow state**: only `dispatch_report_created` may advance to `out_for_delivery`.

# v1.3.0 Supplier Portal Completion — Business-Model-Driven Plan

**Baseline:** v1.2.1 on `main`  
**Status:** Plan only — not implemented

---

## Your business model (from repo docs)

| Actor | Role |
|-------|------|
| **Lokkisona / Sonamoni** | Sells online; owns customer, courier, sync, accounts, settlement |
| **Iqbal & Brothers** | Supplier — packs/ships supplier-handled orders only |
| **IBS-LK ERP** | Supplier-handled fulfillment + **product cost payable** (not selling revenue) |

**Money flow (official ERP):**

```
Orders → Shipped → Owner Dispatch Report → LOCKED cost snapshot
       → Owner posts payable → Ledger → Settlement → Payment to supplier
```

Payable = **dispatch product cost**, never OpenCart selling price or live cost.

**Separate from ERP (engagement tools):**

- **Calculator** — shop helper, no DB
- **Quick Invoice** — supplier’s own walk-in/phone sales to customers; **does not** hit Lokkisona payable unless owner posts separately

**Supplier must NOT:** create dispatch, post payables, run settlements, sync orders, see all 15 owner reports, or manage Lokkisona ecommerce revenue.

**Supplier should:** fulfill orders, confirm returns, see **their balance with Lokkisona** (read-only), see **their shop quick-invoice sales**, use working calculator/invoice tools.

## Decisions (based on business model)

### 1. Dashboard summary — **YES, fulfillment-first**

Not a copy of owner “revenue” dashboard. Supplier KPIs:

| KPI | Source | Why |
|-----|--------|-----|
| Active orders | `countByStatuses` | Daily work queue |
| New / Packaging / Shipped | per-stage counts | Pipeline clarity |
| Pending returns | return receive pending | Action required |
| Balance with Lokkisona | read-only net payable | Trust / accounts visibility |
| Shop invoices (month) | quick_invoice COUNT | Independent sales tool |

Plus compact **Needs attention** + existing task grid as drill-down.

### 2. Accounts — **Read-only Supplier Statement (not full Payables)**

**Page:** `GET /supplier-accounts`  
**Permission:** `supplier_accounts.view` (supplier only)

Shows for **Iqbal & Brothers only**:

- Net payable (ledger running balance)
- Last payment / last dispatch payable (if data exists)
- Recent posted ledger rows (read-only, max 20)
- Safety strip: read-only; disputes via Lokkisona owner

**Does NOT:** post drafts, approve entries, opening balances, settlements.

This matches: *Lokkisona manages accounts; supplier sees their statement.*

### 3. Sales reports — **Two sections, not owner /reports**

**Page:** `GET /supplier-sales`  
**Permission:** `supplier_sales.view` (supplier only)

**A) Fulfillment activity (not “revenue”)**

- Orders shipped (period)
- Returns confirmed / pending
- Optional: dispatch batches including their orders (count only)

No selling price, no Lokkisona ecommerce revenue.

**B) Shop sales (Quick Invoice)**

- Today / month invoice count and total BDT
- Recent quick invoices with reprint while access open
- Label clearly: *Independent shop invoices — not Lokkisona online orders*

### 4. Navigation — expand to 6 links for supplier

| Link | Purpose |
|------|---------|
| Dashboard | Summary + tasks |
| Orders | Workflow |
| Returns | Return receive |
| Accounts | Read-only statement |
| Sales | Fulfillment + shop invoice summary |
| Tools | Calculator / quick invoice entry |

Owner/admin nav unchanged.

### 5. Calculator keyboard — **YES, fix**

Add `keydown` handler in `supplier-tools.js` when calculator modal open.

### 6. Quick Invoice fixes — **YES**

| Fix | Business reason |
|-----|-----------------|
| Save `invoice_date` from form | Shop invoice date must match paper |
| Clear WriteGate message (0007+0010) | Supplier can’t generate if migrations missing |
| Client validation before submit | Fewer failed posts at shop counter |
| Reprint UX hint | One-time access is policy; make it obvious |

Keep one-time supplier print access (audit rule unchanged).

# v1.9.5 — Vendor Fulfillment Workflow Release Foundation Handoff

**Version:** 1.9.5  
**Release label:** Vendor Fulfillment Workflow Release Foundation  
**Route:** `/order-workflow`  
**Baseline:** v1.9.3 Vendor Fulfillment IBS-LK Parity + v1.9.4 Navigation Cleanup  

## Summary

Release-ready vendor fulfillment workflow for first operational testing. Completes the local IBS chain from New Order through Created Report with row and bulk actions, confirmation modals, immutable dispatch batches, audit logging, and read-only OpenCart status display — no OpenCart writes, no page-load API, no new workflow stages.

## Operational flow

| Stage | Row action | Bulk action |
|-------|------------|-------------|
| New Order | Receive Order | Bulk Receive Order |
| Order Received | Start Packaging | Print & Start Packaging |
| Packaging | Mark Shipped | Mark as Shipped |
| Shipped | Create Dispatch Report | Create Dispatch Report |
| Created Report | View Report only (locked) | — |

## Status cards (release UI)

**Main:** New Order · Order Received · Packaging · Shipped · Created Report  

**Exception chips:** Delivery Stop · Hub Return · Customer Return  

Hold and Cancel remain toolbar actions only (not status cards). Out For Delivery and Delivered are not shown on this release page.

## Modals

1. **Receive Order** — generic confirm (`vfActionModal`)
2. **Print & Move to Processing** — bulk packaging with invoice checkbox
3. **Mark Shipped** — confirm + staff checkbox (bulk); row confirm for single
4. **Dispatch Report Created** — report no, totals, Open Dispatch Report button

## Dispatch report rules

- Batch reference: `DDMMYYYY-P1`, `DDMMYYYY-P2`, …
- Locks selected shipped rows; immutable cost snapshot
- Does not update OpenCart
- Payable draft created when ledger table exists; warning appended on success if payable hook fails

## Performance

- Default 20 rows, max 50 per page
- No auto refresh, no background loops

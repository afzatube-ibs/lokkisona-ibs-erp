## Gold-standard pattern applied to every operations page

1. page-header with live `v<?= e($appVersion) ?>` (no hardcoded versions anywhere now)
2. compact one-line safety strip (`ops-safety-strip`) instead of a full Safety Badges card
3. primary operational UI first (forms, tables, actions)
4. ALL developer/planning/raw inventory inside `<details class="planning-collapsible">`, collapsed
5. empty states use `class="empty-state"`
6. no "future plan" cards for features that already exist

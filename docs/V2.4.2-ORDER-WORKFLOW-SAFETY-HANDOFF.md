# v2.4.2 Order Workflow Safety ‚Äî Handoff

**Version:** 2.4.2  
**Status:** Build complete ‚Äî stop before commit  
**Supersedes:** v2.5.0 Supplier Ledger (stopped ‚Äî ledger requires correct order flow first)

## Why this build exists

Ledger and payable posting cannot be trusted until OpenCart sync and IBS workflow boundaries are correct. v2.4.2 enforces channel-neutral order safety: OpenCart feeds orders **up to Shipped**; IBS owns everything after.

`docs/IBS-LK_Workflow_Spec.md` was not present in the repo ‚Äî rules below match the v2.4.2 build brief and legacy Vendor Fulfillment extension (`ibs_lk_supplier_management_erp_v5.8.3`).

## Safety rules implemented

| # | Rule | Implementation |
|---|------|----------------|
| 1 | OpenCart sync stops at Shipped | `OrderSyncWorkflowBoundary::SHIPMENT_CEILING` |
| 2 | Status mapping imports only up to Shipped | `OrderSyncMappingRules` + mapping UI copy |
| 3 | After Shipped, IBS owns workflow | Snapshot-only re-sync; `maybePromoteCourierStage()` forward-only after dispatch |
| 4 | Dispatch Report creation freezes order | `DispatchReportWriteService` + `recordDispatchInclusion()` |
| 5 | Pre-dispatch Hub Return (no payable/ledger/report) | `Shipped ‚Üí Delivery Stop ‚Üí Hub Returning ‚Üí Hub Return Confirmed` |
| 6 | Post-dispatch courier + customer return | `Dispatched ‚Üí In Review ‚Üí In Transit ‚Üí Out For Delivery ‚Üí Delivered` OR `Out For Delivery ‚Üí Customer Returning ‚Üí Supplier Confirmed ‚Üí Return Report` |
| 7 | Block backward movement after dispatch | `OrderWorkflowWriteService::isDispatchImmutable()` + `OrderCourierWorkflowLane` |
| 8 | Block Hub Return after dispatch | Workflow transition guard + `ReturnReceiveWriteService` + UI menu hide |
| 9 | Dispatch eligibility | Shipped + mapped `product_id` + cost > 0 + not hub-return path + not demo |
| 10 | Manual sales | `manual_order_reference` required; `sync_source=manual`; sync never overwrites workflow |
| 11 | Demo/test orders | Hidden by default (`?show_demo=1`); blocked from dispatch batch |

## Workflow paths

### Supplier phases (IBS actions)

```
New Order ‚Üí Order Received ‚Üí Packaging ‚Üí Shipped
```

### Pre-dispatch hub return (TYPE 1 ‚Äî closure only)

```
Shipped ‚Üí Delivery Stop ‚Üí Hub Returning ‚Üí Hub Return Confirmed
```

- Receive on Return List records history only  
- **No** dispatch report, payable, ledger, or return report batch

### Create Daily Dispatch (financial lock)

```
Shipped ‚Üí Create Daily Dispatch ‚Üí Dispatched (dispatch_report_created)
```

- Locks `product_cost_snapshot` per order in `ibs_dispatch_report_items`  
- Payable draft may be created (existing v0.4.7 path) ‚Äî **no auto ledger posting in v2.4.2**

### Post-dispatch courier lane (sync forward-only)

```
Dispatched ‚Üí In Review ‚Üí In Transit ‚Üí Out For Delivery ‚Üí Delivered
```

- `OrderCourierWorkflowLane::forwardPromotionTarget()` on re-sync snapshot refresh  
- Never moves backward

### Post-dispatch customer return (TYPE 2 ‚Äî Return Report)

```
Out For Delivery ‚Üí Customer Returning ‚Üí Supplier Confirmed (Return List) ‚Üí Return Report
```

- `OrderFulfillmentPolicy::shouldCreateReturnReport()` requires dispatch item

## Manual validation checklist

1. **Mapped OC order syncs** ‚Äî Status Mapping ‚Üí Test Order Sync ‚Üí import at mapped status only (‚â§ Shipped)  
2. **Shipped by mapping/action** ‚Äî Map Processing ‚Üí Shipped OR Mark as Shipped  
3. **Pre-dispatch Hub Return** ‚Äî Shipped ‚Üí Delivery Stop ‚Üí Hub Returning ‚Üí Hub Return Confirmed ‚Üí receive ‚Üí **no** return report batch  
4. **Create Dispatch locks** ‚Äî Shipped batch ‚Üí Dispatched ‚Üí cannot revert to Packaging/Shipped  
5. **Post-dispatch return** ‚Äî After dispatch ‚Üí Customer Returning ‚Üí receive ‚Üí return report with dispatch snapshot  
6. **Hub return blocked post-dispatch** ‚Äî Workflow and Return List reject hub path  
7. **Demo blocked** ‚Äî Demo sync order cannot enter dispatch batch  
8. **Re-sync existing** ‚Äî OC status change on dispatched order ‚Üí IBS `ibs_status` unchanged (snapshot fields update; courier may promote forward only)  
9. **Manual order protected** ‚Äî Manual/sales order not reset by sync import

# v1.9.2 Vendor Fulfillment Workflow Foundation ŌĆö Handoff

## Summary

`/order-workflow` is now **Vendor Fulfillment**: a compact white SaaS table loaded from local ERP order snapshots only. Product cards show **supplier cost only** (selling price hidden). IBS fulfillment actions update local workflow status only ŌĆö **OpenCart order status is read-only display**.

| Method | Path | Purpose |
|--------|------|---------|

| POST | `/order-workflow/create` | Manual order create (unchanged) |

## Workflow rules (unchanged semantics)

- Status machine: `OrderWorkflowStatus` + `OrderWorkflowWriteService`
- No move back to New Order
- Created Report / batch-locked rows: no normal fulfillment actions
- Dispatch batch: `DispatchReportWriteService::createDailyBatch()` ŌĆö DDMMYYYY / DDMMYYYY-Pn, cost snapshot, payable draft when ledger ready
- OC Order Status: parsed from import history note (`Source status: ŌĆ”`) until migration 0014 `source_order_status` applied

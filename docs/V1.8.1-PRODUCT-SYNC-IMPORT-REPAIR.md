# v1.8.1 — Product Sync Import Repair

## Goal

Make product sync/import work end-to-end before order import.

## Flow

1. `/sync-preview` → Load product preview (max 20/page)
2. Preview shows Dispatch Location bridge products only (`from_warehouse = 1`)
3. Parent + option/variant rows when available
4. Owner confirms **Import Products**
5. Rows upsert into `ibs_products` / `ibs_product_variants`
6. `/product-control` lists the same tables
7. Re-import updates platform fields only (name, OC model/stock, image, option lines)
8. Re-import never overwrites supplier model, cost, stock, note, or active/inactive

## Fixes in this build

| Area | Change |
|------|--------|
| Write gate | `WriteGate::productSyncImport()` — product tables only, not order sync tables |
| Session | Preview stores bridge-filtered import rows with `from_warehouse = 1` |
| Import | Capped at 20; per-row try/catch; result message: Products imported / Variants imported / Skipped |
| Columns | `image_path` gated like other optional columns |
| UI | Product sync section uses product gate; order sync keeps full gate |

## Safety (unchanged)

- No OpenCart write
- No manual product create
- No product remove
- No order sync changes
- No dispatch/return/payable/invoice changes
- No migration in this build
- No background sync / retry loop

## Manual test

1. Dev DB Activation — apply product migrations (`0003`, variants, business sources)
2. `/sync-preview` → Load product preview (demo mode OK)
3. Confirm 4 demo warehouse products (502 shop-only excluded)
4. Check import confirmation → expect success flash with counts
5. `/product-control` → synced products visible with OC IDs
6. Edit supplier fields in Product Control, re-import → supplier fields unchanged

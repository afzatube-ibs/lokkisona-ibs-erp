## What changed

### v1.4.1 — Product Control UI

- Main **Products** catalog table on `/product-control` (supplier-scoped for supplier login).
- Inline **Edit** form posts to existing `POST /product-control/product/edit`.
- **Create** form adds OpenCart **source product ID** and business source ID.
- Read-only platform fields shown on edit when set: `source_model`, `source_stock`, `last_synced_at`.
- Foundation notes updated: Dispatch Location rule, order-only sync today, supplier access since v1.3.0.

### v1.5.0 — Sync identity wiring (extend existing stack)

- `ProductWriteRepository::findBySourceProductId()` — single lookup helper.
- `SyncImportWriteService` resolves OpenCart line `product_id` → ERP `ibs_products.product_id` via `source_product_id` + `business_source_id`.
- `order_items.source_product_id` stored on import.
- Optional warehouse product pull (same `OpenCartReadClient` + `config/opencart.php`):
  - `product_api_route` — empty disables pull button.
  - `POST /sync-preview/pull-warehouse-products` — upserts `from_warehouse=1` rows only; does not overwrite supplier cost/stock.

### Migration 0011

Apply `database/migrations/0011_supplier_product_category.sql` on staging/production before category column use.

## Operator guide

1. In OpenCart admin, mark supplier products **From Warehouse = Yes** (Dispatch Location).
2. On Sync Preview, run **Pull warehouse products** (sync-only — no manual product create in ERP).
3. On Product Control, edit supplier model, sale/cost, vendor stock, category, status, and note (note requires migration 0012).
4. Run Test Sync + import on Sync Preview (status mapping gate). Line items resolve cost via mapped ERP product.
5. When live OC product API route is known, set `product_api_route` in `config/opencart.php` and re-pull as needed.

## What changed

### Two-step sync (preview → confirm → import)

- **Products:** Load preview (20/page) → owner checkbox → Import/Update ERP
- **Orders:** Load preview (20/page) → owner checkbox → Import eligible only
- **No silent pull** — direct warehouse pull disabled

### Dispatch Location bridge (required)

Product sync includes only rows where `oc_dispatch_location_product.from_warehouse = 1`.

If the bridge is missing on live API response:

> Dispatch Location bridge not found. Product sync cannot safely identify IBS supplier products.

No fallback to all `oc_product` rows.

### Order rules

- Supplier-handled only (`workflow_group` in config allowlist)
- Skip Missing / status 0
- Product mapped in ERP does **not** auto-import orders
- Courier status + consignment stored on import (`courier_status`, `tracking_number`)
- IBS workflow not overwritten on duplicate/re-import

### Product preview fields (read-only unless noted)

Parent: OC ID, name, image, model, price, status, stock, sync status  
Options: option IDs, name/value, model, price/prefix, stock, required, subtract, image

ERP editable after import: supplier model, cost, vendor stock, warning, note, status — unchanged from v1.7.0.

## Purpose

Daily Dispatch Statement is a **supplier payable snapshot** — not a courier/tracking document and not mixed with Returns.

| Field | Accounting meaning |
|-------|-------------------|
| Rate | Locked supplier cost per unit at batch creation |
| Line Total | Qty × Rate |
| Total Dispatched Amount | Amount payable to supplier for this batch |

Print opens **only** when the user clicks **Print Statement** or the Print button on the print page — no auto popup.

## Return module (independent — NOT in v2.3.0)

Returns are **not** shown on Daily Dispatch Statement. Future flow (document only):

1. Hub Return / Customer Return
2. Supplier confirms receipt
3. Return section lists confirmed returns
4. User selects returns → **Create Return Report**
5. Return Report total → **Return Adjustment**
6. Posted to Supplier Ledger as deduction

**Do not build return ledger posting in v2.3.0.**

---

## Supplier ledger future rule (document only)

| Document | Ledger effect |
|----------|---------------|
| Dispatch Report / Statement | Increases supplier payable |
| Return Report | Decreases supplier payable (adjustment) |
| Payments | Decrease supplier payable |

**Net Payable** = Dispatch Total − Return Adjustments − Payments

No automatic ledger posting from dispatch or return pages in v2.3.0.
